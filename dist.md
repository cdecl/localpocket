# LocalPocket Chrome Extension Distribution Guide

This document explains how to package and distribute the LocalPocket Chrome extension.

## 1. Update the Extension Version

Before creating a release ZIP, update the version in `manifest.json`.

Example:

```json
"version": "1.1"
```

Chrome Web Store uploads should use a new version number each time you publish an update.

## 2. Create the Distribution ZIP

From the project root, run:

```bash
./package.sh
```

The script reads `name` and `version` from `manifest.json` and creates a ZIP file in the `dist/` directory.

Current example output:

```bash
dist/localpocket-v1.1.zip
```

## 3. Files Included in the ZIP

The packaging script includes only the files needed for the extension:

- `manifest.json`
- `popup.html`
- `popup.js`
- `style.css`
- `icon.png`
- `images/`
- `lib/`

It does not include development notes or markdown files such as `README.md`, `work_log.md`, or `walkthrough.md`.

## 4. Test Before Upload

Before distribution, verify the packaged extension locally:

1. Open `chrome://extensions`
2. Enable `Developer mode`
3. Click `Load unpacked`
4. Select the project root folder
5. Confirm the popup opens and key features work:
   - Save current tab
   - Save all tabs
   - Edit title
   - Delete item
   - Import/export JSON

If you want to test the exact packaged contents, unzip the file from `dist/` into a temporary folder and load that folder as an unpacked extension.

## 5. Upload to Chrome Web Store

When ready to publish:

1. Open the Chrome Web Store Developer Dashboard
2. Create a new item or open the existing LocalPocket listing
3. Upload the generated ZIP from `dist/`
4. Fill in the store listing fields:
   - Extension name
   - Short description
   - Detailed description
   - Screenshots
   - Icon
   - Category
5. Review permissions shown from `manifest.json`
6. Submit for review and publish

## 6. Developer Dashboard Upload Procedure

Use the Chrome Web Store Developer Dashboard to upload the ZIP generated by `./package.sh`.

### First release

1. Open the Chrome Developer Dashboard
2. Sign in with your Chrome Web Store developer account
3. Click `Add new item`
4. Click `Choose file`
5. Select the generated ZIP file from `dist/`
   - Example: `dist/localpocket-v1.1.zip`
6. Click `Upload`
7. If the ZIP and `manifest.json` are valid, the dashboard opens the item edit page
8. Complete the store listing and required review information
9. Click `Submit for Review`
10. Choose whether to publish immediately after approval or defer publishing

### Updating an existing extension

1. Open the existing item in the Chrome Developer Dashboard
2. Increase the version in `manifest.json`
3. Run `./package.sh` again to create a new ZIP
4. Upload the new ZIP for the existing item
5. Update any listing details that changed
6. Submit the updated version for review

### Important dashboard notes

- The uploaded ZIP must contain `manifest.json` at the top level
- Google documentation states extension packages over `2GB` are rejected
- After upload, review metadata such as privacy declarations, distribution settings, and test instructions if requested
- If you find a manifest typo after upload, fix `manifest.json`, increase the version, create a new ZIP, and upload again

## 7. Release Checklist

- `manifest.json` version updated
- `./package.sh` executed successfully
- ZIP file created in `dist/`
- Extension tested in Chrome
- Store description and screenshots prepared
- Upload completed with the correct ZIP

## 8. Notes

- Keep the ZIP contents flat at the root level. `manifest.json` must be at the top level of the ZIP.
- Do not upload the whole repository as a ZIP.
- Create a new ZIP each time the extension version changes.
